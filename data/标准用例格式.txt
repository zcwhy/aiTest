##一个通用的模板：
{
    "caseNo":测试用例的唯一标记;
    "caseTitle":概述测试用例的主要内容，明确该测试用例的意图;
    "caseModle":所属项目或模块
    "casrPriority":表示用例的重要程度或者影响力，P1~P5（P5最高）
    "casePre":测试用例顺利执行的前提条件，如一些基本的配置（5步以内）;
    "caseData":测试时使用的测试数据（没有就不写这行）;
    "caseStep":如何执行这个测试用例，每步的操作是什么（只能6步以内，如果某一个用例步骤太多请把该用例按更细致的功能点拆分）;
    "caseExpected":和测试步骤对应起来，操作后希望系统的返回（和每一步测试步骤相对应，切忌出现步骤2的结果写在预期结果1里的这种情况）;
}
##一个测试用例格式示例：
{
    "caseNo":001
    "caseTitle":相册大图页收藏功能
    "caseModle":相册-大图页-收藏
    "casrPriority":P5
    "casePre":[
        相册中有图片,    
    ]
    "caseStep":[
        进入相册，打开任意图片进入大图页，点击收藏按钮,
        进入收藏相册查看,
        返回步骤1中的大图页，再次点击收藏按钮,
        重复步骤2,
    ]
    "caseExpected":[
        收藏按钮变红,
        收藏相册中存在步骤1的图片,
        收藏按钮恢复默认样式（不变红）,
        收藏相册中没有步骤1的图片,
    ]
}
##一个测试用例格式示例：
{
    "caseNo":005
    "caseTitle":私密相册动态照片换机
    "caseModle":换机-数据选择-私密相册
    "casrPriority":P5
    "casePre":[
        旧手机上存在私密相册的数据(数据包含动态照片),
    ]
    "caseStep":[
        有私密相册数据的作为旧手机与新手机连接,
        在旧手机的系统数据中选择私密数据，点击下一步,
        勾选数据选项，点击开始传输,
        点击前往验证,
        点击取消，输入隐私密码,
        换机完成,
    ]
    "caseExpected":[
        进入温馨提示页,
        页面下方跳出弹窗“隐私空间验证 您将传输隐私空间中的内容至新设备，请输入隐私密码验证身份”,
        进入输入隐私密码的页面，页面下方跳出隐私密码忘记密码设置 建议绑定小米账号，忘记密码时可通过小米账号重置,
        开始传输接受数据,
        传输完成后，新手机恢复旧手机的私密数据，无丢失，但是旧手机私密相册的密码不会被换过去，需要重新设置,
    ]
}
